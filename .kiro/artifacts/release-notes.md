# Release Notes

## Utilidade

Documenta o que foi publicado em produção: versão, mudanças incluídas, status do deploy. Serve como histórico rastreável de entregas.

## Layout de Documentação

```markdown
# Release <versão>

Owner: devops
Date: YYYY-MM-DD

## Mudanças incluídas
- <referência à story/task> — <resumo>
- ...

## Ambiente
<produção | homologação>

## Status
sucesso | rollback

## Observações
- ...
```

## Caminho do Arquivo

`doc/releases/release-<versão>.md`
